# `context/ui/pages/` — per-page UI/UX narratives

These files are `context/UI_UX.md` §2 ("Pages — narrative") split one file per
page/surface, so UI work loads only the screen in hand.

## Routing rule

When building or reviewing a screen, read **only the page file for that
surface**, plus the cross-cutting parts of `../../UI_UX.md`:

- §0 Design intent and the BLDGTYP design system
- §1 Common elements (header, modals, toasts, empty states, **§1.7 DataTable
  model**, §1.8 evidence/status grammar)
- §4 State-indicator cheatsheet

Do not load every page file. The linked page index lives in `../../UI_UX.md` §2.

## Editing

Keep page files narrative-only (layout, content, behavior — no pixel specs).
Design-system tokens and component-styling rules live in `../../UI_UX.md` §0 and
`frontend/src/styles/README.md`. If you add or rename a page, update the index
table in `../../UI_UX.md` §2.

# Working in `frontend/`

Vite + React + TypeScript app. `src/styles/README.md` is the canonical styling
guide; this file is the rules digest. Repo-wide rules and the closeout gate are
in `../CLAUDE.md`.

## Read before writing frontend code

- `../context/CODING_STANDARDS.md` (Frontend TypeScript) — feature-first
  organization, state ownership, module-size.
- For a screen: the relevant `../context/ui/pages/<page>.md` **plus**
  `../context/UI_UX.md` §0 (design intent) and §1 (common elements, incl. the
  DataTable model §1.7).
- For styling: `../context/UI_UX.md` §0 (design system) and `src/styles/README.md`.

## Non-negotiables

- **`pnpm` only** (never npm/yarn). Keep supply-chain protections on (24h
  `minimumReleaseAge`, strict min-age, `blockExoticSubdeps`). Run
  `pnpm run format` after changes.
- **Display only** — all calculations and data manipulation live in the backend.
- **Plain CSS on the 3-tier token system** — no Tailwind, no shadcn/ui, no
  CSS-in-JS. Feature/shared CSS consumes `var(--token)`; raw hex literals are
  allowed only in the token-definition files (guard-enforced). Radix UI
  provides unstyled behavior primitives (modals, popovers).
- **Server state in TanStack Query**; tables on TanStack Table; client state in
  Zustand. Keep `App.tsx` and route files as composition surfaces, not feature
  logic. Feature-first folders under `src/features/`.
- **DataTable basic affordances are parent-owned and uniform** (iron-law) — see
  `../context/UI_UX.md` §1.7 and
  `../context/technical-requirements/data-table.md`.

## Verifying in a browser

Use the Playwright MCP / `webapp-testing` skill on `http://localhost:5173`, signed
in as `codex@example.com` (never Ed — single-active-session rule). Full setup is
in `../context/ENVIRONMENT.md`. For Model Viewer issues, read
`../planning/archive/model-viewer/AGENT_BROWSER_NOTES.md` first.

## Checks

`make frontend-dev-check` (fast, no DB) for CSS/layout/positioning; for logic,
`cd frontend && pnpm exec vitest run <test-file>`. The full gate before
reporting done is the closeout gate in `../CLAUDE.md`.

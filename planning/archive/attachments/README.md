# Attachments Planning

This folder holds the tracked planning packet for fixed attachment
cells and the shared upload / preview / detach pipeline.

Read order:

1. `STATUS.md`
2. `context/technical-requirements/attachments.md` for the stable
   implementation contract.
3. `phases/testing-verification-strategy.md` for accepted evidence and
   remaining verification work.
4. `PRD.md` for the decision archive and AirTable precedent.
5. Phase files under `phases/` for implementation-plan history.

The stable implementation contract lives in
`context/technical-requirements/attachments.md`.

Do not infer current completion from the `STATUS:` headers in the phase
files. They were written as proposed implementation plans; this folder's
`STATUS.md` is the current feature ledger.

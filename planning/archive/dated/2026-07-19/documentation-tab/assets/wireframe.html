Wireframe v2 — changes from v1 + annotations
  1. 1 — Directions moved into each category: a "📖 How to photograph — X" button per section header opens a category-specific modal (walls show wall shots, ventilators show ventilator shots). Never expanded by default; explicit numbered shot-list per record type.
  2. 2 — Envelope dedup: one row per unique material per assembly (cellulose in 3 segments = 1 row; upload fans out to all its segments in that assembly — same as Materials-page use-site grouping).
  3. 3 — Unified 3-column row grammar everywhere: Spec · Photos · Datasheet, each of Photos/Datasheet with an editor-only "not required" checkbox (the per-axis waiver). Checked → cell renders as muted "not required ✓" and counts as satisfied in the rollup.
  4. 4 — Spec = N/A still disables both axes wholesale (row goes muted).
  5. 5 — Heat Pumps keep leaf sub-heads inside one section (Status-tab disclosure parity).
  6. 6 — Record modal simplified: attributes + evidence read-only (waivers now live inline in the grid, so the modal no longer carries them). Exact per-table attribute list still an open item.
  7. 7 — Envelope datasheet cells are material-level projections: one datasheet array per material, project-wide. Supplying the cellulose datasheet from any assembly row updates every assembly that uses cellulose instantly (same record). Photos remain per material-per-assembly. The material-primary counterpart view already exists as the Envelope → Materials sub-tab.
Toggle Editor/Viewer (top-right): viewers lose selects, drop zones, and waiver checkboxes; keep thumbnails, chips, 📖 directions, and the record modal.
